Spotify Song Popularity Predictor

Built a machine learning model to predict whether a song will become popular on Spotify using audio features and metadata from over 580,000 tracks.

The dataset included features such as danceability, energy, tempo, loudness, instrumentalness, valence, and release year. Songs were labeled as popular if their Spotify popularity score was 60 or higher.

  • Analyzed a dataset of 580,000+ songs from Spotify
  • Engineered features such as release decade and encoded categorical metadata
  • Trained models including Logistic Regression, Random Forest, and XGBoost
  • Converted popularity score into a binary classification problem
  • Evaluated models using F1-score and ROC-AUC with Stratified K-Fold Cross Validation

The best performing model achieved an F1-score of 0.81 when predicting song popularity.

Elevator Simulation

Built a Java based elevator simulation that models how multiple elevator types respond to passenger requests over time. The system supports standard, freight, penthouse, and express elevators, each with its own movement and service constraints.

The simulator processes calls round by round using either a simple scheduler or a complex scheduler. Each round, the scheduler assigns calls, elevators update direction, move one floor at a time, and handle passenger pickup and drop-off.

  • Designed an object-oriented system using abstraction, inheritance, and polymorphism
  • Implemented multiple elevator types with unique service constraints
  • Built simple and complex scheduling strategies using a shared Scheduler interface
  • Read elevator calls from CSV files and simulated system behavior round by round
  • Wrote unit tests for schedulers, elevator behavior, and simulator logic using test-driven development

This project strengthened my understanding of object-oriented design, system simulation, Java testing, and building software that can be extended with new behaviors.

Dungeon Daedalus

Built a text-based dungeon adventure game in Java where the player explores a dungeon, fights enemies, earns gold, buys upgrades, uses potions, and survives increasing difficulty.

The project focuses on object oriented design by separating the game into classes with clear responsibilities instead of keeping all logic in one file.

  • Designed enemies using inheritance, abstraction, polymorphism, and encapsulation
  • Implemented different enemy types such as Slime, Zombie, Skeleton, Lich, Minotaur, and Zombie King
  • Used the Strategy Pattern for different attack behaviors
  • Used the Factory Pattern to create enemies and potions
  • Used the Observer Pattern to display game updates cleanly
  • Improved combat, dungeon progression, shop upgrades, potions, unit tests, and documentation

This project strengthened my understanding of design patterns, testing, and organizing larger Java programs.

Technologies: Java, Gradle, JUnit, Checkstyle, PlantUML, GitHub
